Member Data Documentation

◆ coordinate

std::optional<bool> tatami_mtx::WriteMatrixOptions::coordinate

Whether to save the matrix in the Matrix Market coordinate (sparse) format. If false, the matrix is saved in the array (dense) format instead. If unset, defaults to true if the matrix is sparse.

◆ by_row

std::optional<bool> tatami_mtx::WriteMatrixOptions::by_row

Whether to save coordinate matrices in row-major order, i.e., coordinates are sorted by row first and then by column. If false, the matrix is saved in column-major order, i.e., column first and then row. If unset, defaults to true if the matrix prefers row access.

◆ banner

bool tatami_mtx::WriteMatrixOptions::banner = true

Whether to write the Matrix Market banner. This can be disabled to, e.g., write a custom banner.

◆ format

std::optional<std::chars_format> tatami_mtx::WriteMatrixOptions::format

◆ precision

std::optional<int> tatami_mtx::WriteMatrixOptions::precision

Precision of floating-point values when converted to a string. If unset, the default behavior of std::to_chars() is used.

◆ buffer_size

std::size_t tatami_mtx::WriteMatrixOptions::buffer_size = sanisizer::cap<std::size_t>(65536)

Size of the buffer in which to cache written values before flushing them to the byteme::Writer instance. Larger values improve efficiency at the cost of increasing memory usage.

◆ num_threads

int tatami_mtx::WriteMatrixOptions::num_threads = 1

Number of threads for counting the number of structural non-zeros in a coordinate matrix. This should be positive. Note that this does not affect the writing itself, which is still done in serial.
